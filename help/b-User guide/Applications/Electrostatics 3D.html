3.2. Electrostatics 3D

□ What MaxFEM Solves

The MaxFEM Electrostatics 3D application numerically solves electrostatics bidimensional problems in Cartesian coordinates. You can find extensive information about the electrostatic mathematical model in the MaxFEM Models user guide.

MaxFEM allows to consider linear dielectric materials or conducting media as well as different kinds of charge densities. The admissible options both in the 2D and 3D cases are summarized in the tables below.

Material properties (ε) 2D 3D
Linear and isotropic Yes Yes
Linear and orthotropic Yes Yes

Admisible charges Cartesian 2D Cartesian 3D
Point charges -- Yes
Linear charges Yes Yes
Surface charges Yes Yes
Volumetric charges Yes Yes

Remark: See MaxFEM Electrostatics 2D user guide for further information about 2D electrostatic problems.

For the numerical approximation of electrostatics problems, MaxFEM uses Lagrange finite elements of order one to compute the electric potential V. Thus the electric field is approximated by piecewise functions in each element of the mesh (triangles in 2D and tetrahedra in 3D).

To use the electrostatic application the user needs:

  1. To provide a finite element mesh of the domain: menu Mesh.

  2. To impose the boundary conditions on the boundaries: menu Boundary Conditions.

  3. To assign materials to the different components of the domain: menu Properties. Materials must be previously selected and/or defined in the Materials database.

  4. To assign source properties: menu Sources.

  5. To solve the problem in direct or remote mode: menu Solver/Run or Solver/Run remote, respectively. The user can stop the computation before it is completed in the label Solver/Stop.

  6. Once the problem has been solved, various quantities and fields based on the potential can be computed. It is also possible to create displays of the results: menu Visualization.

All the previous steps are described in the items below

□ The Mesh

MaxFEM does not include a mesh generator tool so the user must provide its own meshes. In the electrostatic application, mesh files can be in unv or mfm format (see section MaxFEM mesh files for a complete description of the mesh formats). MaxFEM allows P1 Lagrange triangular finite elements for 2D problems and P1 Lagrange tetrahedral finite elements in the 3D case. Mesh may be loaded from the Mesh menu by choosing

Mesh → Open → Choose mesh file → name.unv

Preferably, mesh file must be in unv format, but mfm format is also allowed.

□ The Boundary Conditions

Boundary conditions is a menu entry to define boundary or interface conditions and attach them to specific boundaries. Electrostatics application considers two types of boundary conditions.

Dirichlet boundary conditions

In this case, the value VD of the electrostatic potential on the boundary ΓD must be given. To do that, you need:

  1. Define the boundary condition (for instance, blocking_1)

    Boundary conditions → Dirichlet: electrostatic potential, V (V) → blocking_1 → Add

  2. Define the boundary condition via a constant or a precompiled function

    Dirichlet conditions: blocking_1

    The condition is given by: A constant or a function

  3. Assign the boundary condition to the corresponding boundary reference(s). For instance:

    The condition is given by → A constant → Line references → 1 2 4

  4. If VD takes a constant value, define this value:

    The condition is given by: A constant → Constant value → 0.10

  5. If VD is a precompiled function, select “Function defined by user”:

    The condition is given by: A function → Function name → Function defined by user

    In this case, the user must edit the file modulos_electros.f90 of the folder $INSTALLDIR/sources/electrostatics3d/ of the installing directory and define its own function in the line 676 of the module bloqueo, just after the sentences

    elseif (dir%fun(indice) == 1) then ! 'Function defined by user'
    !'User defined: Function defined by user'

  6. Then, the user must recompilate the solver before running it.

Neumann boundary conditions

In this case, the value -D·n of the surface charge density across the boundary ∂ΩN must be given. To do that, you need to follow the same steps as for the Dirichlet conditions but selecting

Boundary conditions → Neumann: normal component of electric field

  1. If the user defines -D.n as a precompiled function, select "Function defined by user" as follows:

    The condition is given by: A function → Function name → Function defined by user

    In this case, the user must edit the file modulos_electros.f90 of the folder $INSTALLDIR/sources/electrostatics3d/ of the installing directory and define its own function in the line 724 of the module neumann, just after the sentence

    elseif (neu%fun(indice) == 1) then ! 'Function defined by user'
    !'User defined: Function defined by user'

    Then, you must recompilate the solver before running it.

□ The Sources

In the electrostatic 3D application, sources can be:

  • Point charge, ρp (C)

  • Line charge density, ρl (C/m)

  • Surface charge density, ρs (C/m2)

  • Volumetric charge density, ρv (C/m3)

To impose a charge density, the user must perform the following steps depending on the type of charge.

Point charge

  1. Choose the type of charge in the Sources item of the application.

    Sources → Point charge, ρp (C)

  2. Then, define a new charge and press the button Add of the dialog box.

    Point sources: source _1 → Add

  3. Press the name source _1 you have just created. Then a new dialog box appears where you must introduce the coordinates of the point(s) where the charge is imposed and its value(s). Notice that you can introduce an array of points where you can define the corresponding charge value.

Line charge density

  1. Choose the type of charge density in the Sources item of the application.

    Sources → Line charge density, ρl (C/m)

  2. Then, define a new charge and press the button Add of the dialog box.

    Line sources: source _1 → Add

  3. Press the name source_1 you have just created and define the charge via a constant or a precompiled function

    Line sources: source_1

  4. If the source takes a constant value, define its value and the corresponding line reference(s):

    The condition is given by → A constant → Line references → 1 2 4

    The condition is given by → A constant → Constant value → 0.1e-10

  5. If the source is a precompiled function, select

    The condition is given by → A constant → Line references → 1 2 4

    The condition is given by → A function → Function name → Function defined by user

    In this case, the user must edit the file modulos_electros.f90 of the folder $INSTALLDIR/sources/electrostatics3d/ of the installing directory and define its own source function in the line 453 of the module cargacur, just after the sentence

    elseif (cur%fun(indice) == 1) then ! fun_0 == 'Function defined by user'

  6. Then, the user must recompilate the solver before running it.

The field must be written in mff format and contain a value for each of the degrees of freedom of the mesh.

The goal of this section is to familiarize the user with the MaxFEM electrostatics 3D application by means of simple examples. These examples contain the general steps and all the data needed to describe the physics and to solve the given cases.

To reproduce exactly the examples, keep all the options in the different menus at the default settings. It is important to note that in the cases where the analytical solution is provided, this solution corresponding with the default settings; the one corresponding with other options the user may have changed is not necessarily the same.
