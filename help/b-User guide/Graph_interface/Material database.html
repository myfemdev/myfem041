2.6. Material database

All applications included in the package MaxFEM share the same material database. When running the program the user can access the list of materials in the menu bar "Materials database".

You can access the materials database content by clicking Material database → Open → Materials. MaxFEM allows you to add your own materials and delete any of those, you can also reset the database to the installation state by clicking Material database → Reset.

Use

The content of this file is loaded when the interface is opened, and its contents is added at the same level as the rest of the menu, though his name Materials database appears separated from the other elements of the menu, located on the right in the menu bar. If you reload the menu (for example, changing the application), the same materials database is integrated back into the new menu. Then the behavior is the same as if it was part of the menu, just saved in the same file. All applications share de same database.

When a material is defined, you can choose which properties want to complete. These properties can be defined by a constant, by a temperature dependant table or by a function. A function defined material property implies that this function must be implemented in the application source code. The function code is identified by the material name.

After implementing the function is necessary to compile the corresponding application.

Location

Initially, the materials database is located in INSTALLDIR/materialsDB/materials.<name>.xml, where <name> corresponds to the name of the package being used. After first run you work with a local copy of the database, the appropriate file is copied into local MaxFEM directory.

The database itself may include in a submenu, a block named reset_materials, that causes, when selected in the interface, that the original Materials Database is copied again from the installation directory to the directory configuration, and therefore all the materials inserted after the installation are deleted.

	<menus>
		<menu name="Materials database">
			<submenu name="Open">
			(...)
			</submenu>
			<submenu name="Reset">
				<action name="reset_materials">
				</action>
			</submenu>
		</menu>
	</menus>

The interface will only change the copy in the configuration directory, never the one in the installation directory, so you should make changes, you can always return to the initial state.
