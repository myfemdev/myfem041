Folder location

If MaxFEM is installed in the folder INSTALLDIR, the source code is in INSTALLDIR/sources.

MaxFEM searches for the executable solvers in a folder named MaxFEM located in the user's home directory. If the executable file does not exist in that folder, or the folder does not exists, MaxFEM searches in the folder INSTALLDIR/solvers. This behaviour is important when the user wants to:

  • compile the solvers by itself (see Compilation) or
  • execute the solvers remotely (see Run remote)

In any other case, i.e. when MaxFEM is installed through a binary file and solvers are executed locally, the behaviour described above is not relevant.

These are the usual locations of the MaxFEM folder in the user's home directory, depending on the operating system:

  • GNU/Linux: /home/<user>/.MaxFEM.

  • Windows XP: C:\Documents and Settings\<user>\Application Data\MaxFEM.

  • Windows 7: C:\Users\<user>\AppData\Roaming\MaxFEM.

  • Mac OS X: /Users/<user>/Library/Application Support/MaxFEM.

Here <user> stands for the username.

Compilation

MaxFEM solvers can be compiled by the user. The software required to compile is indicated in Software requirements and the instructions to compile them in Building from source.

A user would like to compile the solvers for two reasons:

  • When MaxFEM installation is made from source, executable files do not exist.

  • When the user defines a boundary condition, source, etc. via precompiled functions, changes take effect when the solvers are recompiled.

If you do not have write privileges in the folder INSTALLDIR, you still can save the execuatble solvers in your home directory. In such a case, you must:

  1. Copy folders INSTALLDIR/solvers and INSTALLDIR/sources in the mentioned Folder location.

  2. Enter in folder sources, copied in the mentioned Folder location, and compile the solvers following the instructions indicated in Building from source.
