2.4. MaxFEM field files

All the MaxFEM solvers write the resulting fields in the working folder using the VTK XML format, developed by Kitware Inc. The file type is serial and the mesh is considered an unstructured grid. Thus, the corresponding extension for the result files is VTU. The VTK format is described here.

MaxFEM solvers use a Fortran 90 module to write VTK XML. The main portion of this code has been programmed by Stefano Zaghi (Dept. of Computational Hydrodynamics at INSEAN).

MaxFEM automatically loads the result files. If the user wants to manage and/or visualize the results by its own, a wide range of graphic applications support VTK XML files, like Paraview, MayaVi, EnSight,... For each result field, the name of the VTU file can be seen by clicking Names button in the Visualization menu.

Data fields

In some applications, a data field must be provided by the user. That is the case, for example, when a temperature field is necessary to work with materials with properties that depend on temperature. In such case, VTK XML is a valid format for those data files.

Alternatively, the user can provide a file in MFF format. The files written in this simple format are ASCII files composed of a single record. This record contains an integer nver and a double precision array v, with all the elements separated by blank spaces:

  • nver is the number of nodes of the mesh multiplied by the dimension of the field (a scalar field has dimension 1).

  • v is the array formed by the field values on the nodes. In the case of a vector field, all the coordinates of the field in each node appear consecutive in v.
